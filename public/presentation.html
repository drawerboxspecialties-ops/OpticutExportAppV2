DBS Drawers — OptiCut CSV Splitter

One-page guide · Allmoxy CSV → OptiCut cut lists & shop stack sheets

Live app:
drawerboxspecialties-ops.github.io/OpticutExportAppV2/

The problem

Allmoxy exports one large CSV. OptiCut needs smaller, material-matched cut lists. The shop needs stack sheets with box counts per drawer height and order. This tool runs entirely in your browser — no upload, no server.

What you get
  • Split batches by material + top edge (+ special orders)
  • Cut-list CSV per batch for OptiCut
  • Stack matrix on screen and printable
  • ZIP export of all batch CSVs
End-to-end flow
Allmoxy CSV→Parse→Normalize→Special flags→Batch→Split→Export / Print
1 · Import columns

Core (exported): OrderNumber, MaterialName, PartName, W, Length, Quantity, Label, Width, TopEdge.

Batching-only (not exported): GroupID, Laser, Scoop, Slope, DrillFront, DividersFB, DividersSS, FileSlots.

2 · Normalize
  • Top edges cleaned to standard names
  • B-edge priority: front rows inherit back top edge when matched
  • Material names shortened for OptiCut (≤ 32 chars)
3 · Special orders (default ON)

Order is special if any row has a value other than blank / None in Scoop, Slope, Dividers, DrillFront, or FileSlots. Gets a SPECIAL_ batch — never mixed with normal orders of same material/edge.

4 · Batch grouping

Bucket key: [SPECIAL_] + Category + Edge + Material + TopEdge

CategoryCode
PlywoodPLY
FAAFAA
Solid woodSLD
MDF / PVC / tapeMDF

Each order appears in exactly one split batch.

5 · Box math & 6 · Export

boxes = ceil(parts ÷ 4) — fixed shop rule for stack matrix and batch totals.

Export strips batching columns. With round widths ON: Width rounds up, matching rows merge, Label notes original widths when changed.

Operator checklist

  1. Export CSV from Allmoxy (include extra columns if used).
  2. Open the app → drag & drop the file.
  3. Review batches (watch for SPECIAL badge).
  4. Print stack matrix (current batch or all).
  5. Export CSV or ZIP → import each file into OptiCut.
